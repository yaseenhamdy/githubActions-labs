# Labs Guide

## Purpose

This folder holds the main guided practice path for the course.

Read the labs in order.

Each lab teaches one clear step in the course story.

## Main Guided Path

Read these labs in order first:

1. [LAB-01: First Workflow](LAB-01-first-workflow.md)
2. [LAB-02: Real CI Workflow](LAB-02-real-ci-workflow.md)
3. [LAB-03: Build Artifact Workflow](LAB-03-build-artifact-workflow.md)
4. [LAB-04: Deploy Workflow](LAB-04-deploy-workflow.md)
5. [LAB-05: Full CI/CD Flow](LAB-05-full-cicd-flow.md)

## Later Extensions

Use these only after the main guided path is already complete:

1. [LAB-06: Full Containerized CI/CD Pipeline](LAB-06-full-containerized-cicd-pipeline.md)
2. [LAB-07: Final Assessment Setup and Validation Prep](LAB-07-docker-hub-vm-deploy.md)

## How These Labs Fit the Story

- `LAB-01` introduces workflow structure
- `LAB-02` teaches verification
- `LAB-03` teaches packaging
- `LAB-04` teaches delivery
- `LAB-05` connects the full beginner flow
- `LAB-06` is an optional next-step containerized example after the core path
- `LAB-07` prepares the later final assessment setup after the core path

## One Sequence Rule

Do not read the numbering alone as the course path.

Use this order:

- first complete `LAB-01` to `LAB-05`
- then use `LAB-06` only if your instructor chooses the optional fuller example
- use `LAB-07` only when your instructor starts the Ubuntu VM assessment path
